Array ADT Abstract Data Type

01

What is Array ADT?

📦
Array ADT is an Abstract Data Type that represents a finite, ordered collection of elements — all of the same type — stored at contiguous memory locations and accessed using an integer index.

In the ADT perspective, we define what an array does — not how it is stored internally. An Array ADT specifies a fixed capacity, zero-based indexing, and a set of operations (create, insert, delete, search, traverse) without exposing memory details to the user.

The key distinction: a raw C array is just a block of memory, while the Array ADT wraps it with a size, a maximum capacity, and well-defined operations — turning a primitive into a proper data structure.

02

Real World Analogies

🪑

Seats in a Classroom

Each seat has a fixed number (index). You can directly go to seat 5 without checking every other seat. The classroom has a fixed capacity — just like an array.

📚

Books on a Shelf

Books are arranged in a specific order on a shelf. You access book at position 3 directly. Inserting a book in the middle requires shifting others to the right.

🅿️

Parking Slots

A parking lot has numbered slots (0, 1, 2 …). You know exactly which slot is free and can drive directly to slot 7. The total number of slots is fixed at construction.

03

Types of Arrays

1D Array

A single row of elements stored sequentially. Accessed with one index: arr[i]. Most common and fundamental type.

Access Time O(1)
2D Array

A grid of rows and columns — essentially an array of arrays. Accessed with two indices: arr[i][j]. Used for matrices and tables.

Access Time O(1)
Multi-D Array

Three or more dimensions — e.g. arr[i][j][k]. Used in scientific computing, image processing, and 3D data.

Access Time O(1)

04

Operations on Array ADT

Traversal

Visit every element one by one from index 0 to size−1. Used to print, sum, or process all elements.

Time Complexity O(n)
Insertion

Add an element at a given index. Elements after that position must shift right by one. Costly at the beginning.

Time Complexity O(n)
Deletion

Remove an element at a given index. Elements after it must shift left. Size decrements by one after deletion.

Time Complexity O(n)
Searching

Find an element by value (linear search) or by index (direct access). Index-based access is instant.

Time Complexity O(n) / O(1)

Example Implementation in C

Below is a struct-based Array ADT in C. It encapsulates the data array, current size, and maximum capacity — exposing only named operations to the user.

The ArrayADT struct wraps a fixed-size array with a size tracker and max capacity field.

C — Array ADT Struct 
#include <stdio.h>
#define MAX 10

// Array ADT definition
struct ArrayADT {
    int data[MAX]; // underlying storage
    int size;      // current number of elements
    int capacity;  // maximum capacity
};

// Create / initialise an empty array ADT
void create(struct ArrayADT* arr) {
    arr->size     = 0;
    arr->capacity = MAX;
}
ℹ️ Encapsulation: By hiding the raw array inside a struct, users interact only through functions — the core idea of an ADT.

The four fundamental operations — insert, delete, search, and display — work on the struct pointer.

C — Insert, Delete, Search, Display 
// Insert element at position pos
void insert(struct ArrayADT* arr, int pos, int value) {
    if (arr->size >= arr->capacity) {
        printf("Array is full!\n"); return;
    }
    for (int i = arr->size; i > pos; i--)
        arr->data[i] = arr->data[i - 1]; // shift right
    arr->data[pos] = value;
    arr->size++;
}

// Delete element at position pos
void deleteAt(struct ArrayADT* arr, int pos) {
    if (pos < 0 || pos >= arr->size) {
        printf("Invalid position!\n"); return;
    }
    for (int i = pos; i < arr->size - 1; i++)
        arr->data[i] = arr->data[i + 1]; // shift left
    arr->size--;
}

// Linear search — returns index or -1
int search(struct ArrayADT* arr, int value) {
    for (int i = 0; i < arr->size; i++)
        if (arr->data[i] == value) return i;
    return -1;
}

// Display all elements
void display(struct ArrayADT* arr) {
    printf("Array: [");
    for (int i = 0; i < arr->size; i++) {
        printf("%d", arr->data[i]);
        if (i < arr->size - 1) printf(", ");
    }
    printf("]\n");
}
ℹ️ Shift Cost: Insertion and deletion at the beginning require shifting all elements — O(n) in the worst case.

06

Advantages & Limitations

⚡

O(1) Direct Access

Any element can be accessed in constant time using its index. No need to traverse — ideal for lookup-heavy tasks.

🧠

Cache Friendly

Elements are stored contiguously in memory. Modern CPUs prefetch adjacent data, making array traversal extremely fast in practice.

📏

Fixed Size

Size must be declared at creation time. If you underestimate, you run out of space; if you overestimate, memory is wasted.

🔀

Costly Insertions/Deletions

Adding or removing elements in the middle requires shifting other elements, which takes O(n) time — inefficient for frequent mutations.

07

Applications of Array ADT

💾

Data Storage

Arrays are the backbone of most in-memory data storage. Databases use arrays internally for fixed-schema row storage and column buffers.

📊

Matrices & Grids

2D arrays represent mathematical matrices, game boards, pixel grids, and spreadsheet data — enabling efficient row/column access.

🔎

Lookup Tables

Pre-computed values (trigonometry tables, colour palettes, hash buckets) are stored in arrays for O(1) lookup — trading memory for speed.

🧩

Building Other Structures

Stacks, Queues, Heaps, and Hash Tables are commonly implemented using arrays as their underlying storage mechanism.
