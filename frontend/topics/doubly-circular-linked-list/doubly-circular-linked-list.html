Doubly Circular Linked List Circular – Bi-directional

01

What is a Doubly Circular Linked List?

🔄
A Doubly Circular Linked List combines the properties of a doubly linked list and a circular linked list. Every node has a next pointer and a prev pointer, and the tail's next points back to head while head's prev points back to tail — forming a fully circular, bidirectional structure.

This is the most feature-rich of the linked list variants. It supports traversal in both directions, has no NULL terminators in either direction, and enables O(1) insertions at both ends with a tail pointer. The cost is two pointers per node, doubling pointer overhead compared to a singly linked list.

🎠

Carousel (Both Ways)

You can step forward or backward between horses, and the last horse connects back to the first — both forward and backward.

🔃

Browser History

Navigate forward and backward through pages. In a DCLL implementation, the last page's next wraps to the first.

🎡

Two-Way Ring

Nodes form a ring you can traverse clockwise (next) or counter-clockwise (prev) — never hitting a dead end.

Key Properties

Bidirectional — traverse forward (next) or backward (prev)
Fully circular — no NULL in either direction
Two invariants: tail->next == head AND head->prev == tail
Extra overhead: 2 pointers per node vs 1 in SLL

vs. Singly Circular LL

SCLL: 1 pointer/node  |  DCLL: 2 pointers/node
SCLL: forward only  |  DCLL: forward and backward
DCLL: delete without predecessor traversal — O(1) with node reference
03

Operations

Insert Begin

Create node, link it between tail and current head, update head. Both circular invariants must hold.

TimeO(1)
Insert End

Create node, link it after tail and before head, update tail. Circular invariants restored.

TimeO(1)
Delete Node

Relink predecessor and successor via both next and prev. With a node reference: O(1), otherwise O(n) search.

TimeO(1)/O(n)
Traversal

Forward: do-while with next. Backward: do-while with prev. Both stop when curr == head.

TimeO(n)
04

Implementation in C

The node holds three fields: prev, data, next.

C — Node Structure 
#include <stdio.h>
#include <stdlib.h>

struct Node {
    struct Node* prev;
    int          data;
    struct Node* next;
};

struct Node* head = NULL;
struct Node* tail = NULL;

struct Node* createNode(int val) {
    struct Node* n = (struct Node*)malloc(sizeof(struct Node));
    n->data = val;
    n->next = n->prev = NULL;
    return n;
}

Both next and prev links must be maintained. After every operation: tail->next == head and head->prev == tail.

C — Insert & Delete 
void insertAtEnd(int val) {
    struct Node* n = createNode(val);
    if (head == NULL) {
        head = tail = n;
        n->next = n->prev = n;   // single node — points to itself
    } else {
        n->prev    = tail;
        n->next    = head;
        tail->next = n;
        head->prev = n;
        tail       = n;
    }
}

void insertAtBeginning(int val) {
    struct Node* n = createNode(val);
    if (head == NULL) {
        head = tail = n;
        n->next = n->prev = n;
    } else {
        n->next    = head;
        n->prev    = tail;
        head->prev = n;
        tail->next = n;
        head       = n;
    }
}

void deleteNode(int val) {
    if (head == NULL) { printf("Empty.\n"); return; }
    struct Node* curr = head;
    do {
        if (curr->data == val) break;
        curr = curr->next;
    } while (curr != head);

    if (curr->data != val) { printf("Not found.\n"); return; }

    if (curr == head && curr == tail) { head = tail = NULL; }
    else {
        curr->prev->next = curr->next;
        curr->next->prev = curr->prev;
        if (curr == head) head = curr->next;
        if (curr == tail) tail = curr->prev;
    }
    free(curr);
}
ℹ️ Unlike SLL/SCLL, deletion in DCLL doesn't need a predecessor search — just relink curr->prev->next and curr->next->prev.

Can traverse in both directions using the same do-while pattern, just changing curr->next to curr->prev.

C — Forward & Backward Traversal 
void displayForward() {
    if (!head) { printf("Empty.\n"); return; }
    struct Node* curr = head;
    printf("HEAD → ");
    do {
        printf("[%d] ⇌ ", curr->data);
        curr = curr->next;
    } while (curr != head);
    printf("(HEAD)\n");
}

void displayBackward() {
    if (!head) { printf("Empty.\n"); return; }
    struct Node* curr = tail;
    printf("TAIL → ");
    do {
        printf("[%d] ⇌ ", curr->data);
        curr = curr->prev;
    } while (curr != tail);
    printf("(TAIL)\n");
}

05

Advantages & Limitations

✅

Bidirectional Traversal

Navigate forward and backward without any extra data structures — ideal for undo/redo, browser history.

✅

O(1) Delete (by reference)

Given a node reference, delete it in O(1) by relinking neighbours — no predecessor search needed.

✅

Efficient Queue/Deque

Supports O(1) insert/delete at both ends — perfect for a double-ended queue (deque).

❌

2× Pointer Overhead

Two pointers per node doubles pointer memory vs SLL. For small data types this is significant.

❌

Complex Logic

Every insert/delete must maintain four pointer assignments instead of two. Bugs from broken invariants cause infinite loops.

06

Applications

↩️

Undo / Redo

Text editors use a doubly circular list to maintain action history — navigate back (prev) to undo, forward (next) to redo.

🌐

Browser Tab Cycling

Ctrl+Tab / Ctrl+Shift+Tab cycle through tabs forward and backward — a DCLL models this naturally.

🎯

LRU Cache

Least-recently-used caches use a DLL (often circular) to maintain order — move accessed items to front in O(1).

🎵

Media Player Controls

Previous / Next song navigation with loop mode enabled — a DCLL lets you go forward or backward through the playlist circularly.
