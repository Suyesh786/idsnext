Queue FIFO – First In First Out

01

What is a Queue?

🚶
Queue is a linear data structure that follows the FIFO (First In, First Out) principle. The element inserted first is the first one to be removed — just like people standing in a line.

A queue is an ordered collection of elements where insertion happens at one end (called the Rear) and deletion happens at the other end (called the Front). Think of it as a pipe — whatever goes in first, comes out first. This predictable ordering makes queues essential for scheduling, buffering, and resource management in computer systems.

02

Real World Analogies

🧍

People in a Queue

People join a line at the rear and are served from the front. The person who arrives first gets served first — no cutting allowed!

🖨️

Printer Job Queue

When multiple documents are sent to a printer, they queue up. The first document sent is the first one printed, maintaining order.

🎟️

Ticket Counter

At a ticket counter, customers take a token number. The one with the smallest number is called first — earliest arrival, earliest service.

05

Queue Implementation Concept

In an array-based queue, we maintain two indices — front and rear. Enqueue increments rear and places the element; Dequeue increments front. The downside is wasted space when front keeps moving right.

C 
#include <stdio.h>
#define MAX 100

int queue[MAX];
int front = -1;
int rear = -1;

void enqueue(int value) {
    if (rear == MAX - 1) {
        printf("Queue Overflow\n");
        return;
    }

    if (front == -1)
        front = 0;

    rear++;
    queue[rear] = value;
}

int dequeue() {
    if (front == -1 || front > rear) {
        printf("Queue Underflow\n");
        return -1;
    }

    return queue[front++];
}
💡 In C, queues are typically implemented using arrays or linked lists. Proper index management allows enqueue and dequeue operations to run in O(1) time.

A Circular Queue solves the wasted-space problem of a simple array queue. When the rear reaches the end of the array, it wraps around to the beginning using modulo arithmetic — forming a logical circle. This allows reuse of freed slots efficiently.

C 
#include <stdio.h>
#define SIZE 5

int queue[SIZE];
int front = -1;
int rear = -1;

void enqueue(int value) {

    if ((rear + 1) % SIZE == front) {
        printf("Queue Overflow\n");
        return;
    }

    if (front == -1)
        front = 0;

    rear = (rear + 1) % SIZE;
    queue[rear] = value;
}

int dequeue() {

    if (front == -1) {
        printf("Queue Underflow\n");
        return -1;
    }

    int value = queue[front];

    if (front == rear)
        front = rear = -1;
    else
        front = (front + 1) % SIZE;

    return value;
}
💡 The key trick is (rear + 1) % capacity. When rear hits the last index, it wraps to 0 — reusing slots freed by earlier dequeues. Both enqueue and dequeue remain O(1).

06

Applications of Queue

🖥️

CPU Scheduling

Operating systems use queues to manage processes waiting for CPU time. Ready queues ensure fair, ordered execution of tasks.

🌐

BFS in Graphs

Breadth-First Search uses a queue to explore nodes level by level. Each unvisited neighbor is enqueued and explored in order.

📡

Network Buffering

Data packets are placed in queues when the network is busy. They are transmitted in order, preserving message integrity.

🎮

Event Handling

Games and GUIs use event queues. Mouse clicks, key presses, and touch events are queued and processed one at a time in order.
