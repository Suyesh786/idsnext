Stack LIFO – Last In First Out

01

What is a Stack?

📚
Stack is a linear data structure that follows the LIFO (Last In, First Out) principle. The element inserted last is the first one to be removed — just like a stack of plates.

A stack is an abstract data type with a bounded (or unbounded) capacity. It is a simple data structure that allows adding and removing elements in a particular order. Every time an element is added, it goes on top of the stack. The only element that can be removed is the element that is at the top of the stack, just like a pile of objects.

02

Real World Analogies

🍽️

Stack of Plates

You place plates on top of one another. When you need a plate, you take the top one. You cannot remove a plate from the middle without removing all plates above it.

🌐

Browser Back Button

Every page you visit is pushed onto a stack. When you click "Back", the most recently visited page (top of stack) is popped and you return to the previous one.

↩️

Undo in Editors

Every edit operation is pushed onto a stack. Pressing Ctrl+Z pops the last operation, effectively undoing the most recent change first.

Stack Implementation in C

An array-based stack uses a fixed-size array and a top pointer that tracks the index of the top element.

C — Stack using Array 
#define MAX 100

int stack[MAX];
int top = -1;

// Push element onto stack
void push(int value) {
    if (top == MAX - 1) {
        printf("Stack Overflow!\n");
        return;
    }
    stack[++top] = value;
}

// Pop element from stack
int pop() {
    if (top == -1) {
        printf("Stack Underflow!\n");
        return -1;
    }
    return stack[top--];
}

// Peek at top element
int peek() {
    if (top == -1) return -1;
    return stack[top];
}

// Check if stack is empty
int isEmpty() {
    return top == -1;
}
ℹ️ Space Complexity: O(n) — requires pre-allocated memory for MAX elements. Stack Overflow occurs when top == MAX - 1.

A linked list-based stack uses dynamic memory. Each node stores a value and a pointer to the node below it. The head of the list is the top of the stack.

C — Stack using Linked List 
struct Node {
    int data;
    struct Node* next;
};

struct Node* top = NULL;

// Push element onto stack
void push(int value) {
    struct Node* newNode =
        (struct Node*) malloc(sizeof(struct Node));
    newNode->data = value;
    newNode->next = top;
    top = newNode;
}

// Pop element from stack
int pop() {
    if (top == NULL) {
        printf("Stack Underflow!\n");
        return -1;
    }
    int val = top->data;
    struct Node* temp = top;
    top = top->next;
    free(temp);
    return val;
}

// Peek at top element
int peek() {
    if (top == NULL) return -1;
    return top->data;
}

// Check if stack is empty
int isEmpty() {
    return top == NULL;
}
ℹ️ Advantage: Dynamic size — no overflow unless system runs out of heap memory. Each node is allocated on demand using malloc().

05

Applications of Stack

🧮

Expression Evaluation

Stacks are used to evaluate arithmetic expressions in postfix (RPN) notation, and to convert between infix, prefix, and postfix forms.

📞

Function Call Stack

Every function call pushes a stack frame (local vars, return address) onto the call stack. When a function returns, its frame is popped.

🔍

DFS in Graphs

Depth-First Search uses a stack (explicit or via recursion) to explore nodes. Each unvisited neighbor is pushed and explored depth-first.

🔤

Parenthesis Checking

Push every opening bracket. When a closing bracket is found, pop and verify it matches. If the stack is empty at end, brackets are balanced.
