Expression Tree Expression Representation

01

What is an Expression Tree?

∑
An Expression Tree is a binary tree that stores an expression in tree form. Operators become internal nodes, and operands become leaf nodes.

Instead of reading an expression only as text, an expression tree shows the actual solving structure. The top node is the final operation, and the children are the smaller expressions that must be solved first.

For example, in (A + B) * C, the addition happens before multiplication because it is lower in the tree. The root * is the final operation.

💡 Compilers use tree-like structures to understand expressions, preserve operator precedence, and generate correct machine instructions.
Expression: (A + B) * C
02

Why Do We Need Expression Trees?

Expressions written as strings can be confusing for a computer. Parentheses, precedence, and associativity must all be interpreted correctly. A tree makes the order of work visible.

()

Handles precedence

Multiplication, division, addition, and subtraction are placed according to the real solving order.

↧

Shows dependency

A parent operator waits until its child expressions are evaluated first.

⚙

Helps compilers

Compilers parse expressions into tree form before checking, optimizing, or generating code.

03

Structure of an Expression Tree

Root

The topmost operator. It usually represents the final operation of the expression.

Internal Nodes

These are operators like +, -, *, and /. They combine two child results.

Leaf Nodes

These are operands like variables and numbers. They do not have children.

ℹ️ Most basic expression trees are binary because each operator here works on two operands: left value and right value.
04

Components of an Expression Tree

Expression trees are easy to read once you separate their two node types: operators and operands.

Operator Nodes Internal nodes
+ - * /

Operators perform work. They take values from their left and right children.

Operand Nodes Leaf nodes
A B 5 10

Operands hold actual values. They are the starting points for evaluation.

05

How an Expression Tree Represents an Expression

The tree form of (A + B) * (C - D) makes the expression's grouping visible. The root * says: multiply the result of the left expression by the result of the right expression.

Expression (A + B) * (C - D)

Building an Expression Tree

A common method is to convert an infix expression to postfix, then scan the postfix expression from left to right. Operands become nodes immediately; operators connect the two most recent nodes.

Infix Expression
↓

↓

↓

↓

💡 For (A + B) * (C - D), postfix becomes A B + C D - *. That postfix form is very convenient for building the tree using a stack.
07

Expression Tree Traversals

This is the most important idea: the same expression tree can be read in different traversal orders, and each order gives a different expression notation.

Sample tree: (A + B) * (C - D)
Inorder Traversal Left → Root → Right

Infix: ((A + B) * (C - D))

Human-friendly form, usually with parentheses to preserve meaning.
Preorder Traversal Root → Left → Right

Prefix: * + A B - C D

Operator appears before its operands.
Postorder Traversal Left → Right → Root

Postfix: A B + C D - *

Operator appears after its operands; useful for stack evaluation.
08

Expression Evaluation

Expression trees are evaluated from the leaves upward. First evaluate the children, then apply the parent operator.

Expression: (5 + 2) * 3
1 Evaluate the left subtree: 5 + 2 = 7
2 Use that result at the root: 7 * 3
3 Final answer: 21
Result = 21
09

Expression Tree vs Binary Tree

An expression tree is a special use of a binary tree. The structure is binary, but the meaning of each node is specific to expressions.

Topic Binary Tree Expression Tree
Purpose General hierarchical data storage. Represents arithmetic or logical expressions.
Data Stored Any kind of value. Operators and operands.
Structure Each node can have up to two children. Operators usually have two children; operands are leaves.
Usage Used for general tree algorithms. Used for parsing, conversion, and evaluation.
Traversal Meaning Shows visit order. Produces infix, prefix, or postfix expressions.

13

Implementation in C

The basic C structure is small. Each node stores one character and two child pointers. Operators and operands are stored in the same node type.

C — Node Structure 
struct node {
    char data;
    struct node* left;
    struct node* right;
};
C — Create a Node 
#include<stdio.h>
#include<stdlib.h>

struct node* createNode(char value) {
    struct node* newNode = (struct node*)malloc(sizeof(struct node));
    newNode->data = value;
    newNode->left = NULL;
    newNode->right = NULL;
    return newNode;
}
C — Simple Manual Tree 
struct node* root = createNode('*');
root->left = createNode('+');
root->right = createNode('C');

root->left->left = createNode('A');
root->left->right = createNode('B');
👁️ This creates the tree for (A + B) * C. A full program would usually build this automatically from postfix or another parsed form.
