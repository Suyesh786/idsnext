Tree Representation Array vs Linked List

01

What is Tree Representation?

🌳
A binary tree exists in our mind as a diagram with nodes and branches — but a computer stores data in memory, not as pictures. Tree representation is how we translate that diagram into actual memory. There are two ways to do this: Method 1 — using an Array, and Method 2 — using a Linked List (nodes connected by pointers). Both store the exact same tree, just in different ways. Choosing the right one depends on your situation.
02

Method 1: Array Representation

Storing a tree in an array

We number positions in the array starting from 1. The root always goes at index 1. For any node sitting at index i, we can find its children using a simple formula — no extra pointers needed.

If a position has no node (an empty slot), we store 0 there as a placeholder.

Node at index i
Left child→ 2i
Right child→ 2i + 1
Parent→ ⌊i / 2⌋

→ Node 5 is the right child of node 2
→ Node 2 is at index 2
→ Right child of index 2 = 2×2+1 = index 5 ✓
∴ Node 5 → array[5] = 5
04

Method 2: Linked List Representation

Storing a tree using nodes and pointers

Each node is a struct with 3 fields: data holds the value, left points to the left child, and right points to the right child. If a child doesn't exist, that pointer is NULL.

Nodes are connected by pointers — not by array indices. This is the most common way to implement trees in C.

C 
struct node {
    int data;
    struct node* left;
    struct node* right;
};

Each node stores its value plus two pointers. NULL means no child exists.

06

When to Use Which?

📦

Use Array When

  • Tree is complete or perfect — no missing nodes
  • You need fast random access by level
  • Memory is pre-allocated and fixed
  • Example: Heap data structure
🔗

Use Linked List When

  • Tree is sparse — many positions are empty
  • Tree size changes dynamically at runtime
  • Building a BST, AVL tree, or general tree
  • Example: Binary Search Tree
07

Implementation in C

We implement the Linked List method in C using a recursive create() function. The user enters -1 to indicate no child — the function returns NULL immediately. The left subtree is fully built before asking for the right side.

C 
#include<stdio.h>
#include<stdlib.h>

struct node {
    int data;
    struct node* left;
    struct node* right;
};

struct node* create() {
    int x;
    printf("Enter data (-1 for NULL): ");
    scanf("%d", &x);
    if(x == -1) return NULL;
    struct node* n = malloc(sizeof(struct node));
    n->data = x;
    n->left = create();
    n->right = create();
    return n;
}

int main() {
    struct node* root = create();
}
08

Advantages & Limitations

Array Representation

✔️

Simple index-based access

Find any node instantly with array[i]. No traversal needed.

✔️

No pointers needed

Cleaner code — parent-child relationships come from the 2i formula.

✔️

Cache-friendly

Contiguous memory = faster access. CPU cache loves arrays.

✖️

Wastes memory

Sparse trees leave many array slots empty, wasting space.

✖️

Fixed size

Hard to grow the array dynamically once allocated.

✖️

Skewed trees are expensive

A skewed tree of n nodes can need up to 2ⁿ array slots.

Linked List Representation

✔️

Memory efficient

Allocates only for nodes that exist — no wasted slots.

✔️

Grows dynamically

malloc() creates nodes on demand. Tree can shrink and grow freely.

✔️

Natural for recursion

Recursive algorithms map perfectly to pointer-based node structures.

✖️

Pointer overhead

Each node holds 2 extra pointers — memory adds up for large trees.

✖️

No direct index access

Must traverse from root to reach any node — no O(1) lookup.

✖️

Pointer bugs

Dangling or NULL pointer errors can silently corrupt the entire tree.
