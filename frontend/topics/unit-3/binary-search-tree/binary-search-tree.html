01

What is a Binary Search Tree?

🔎
A Binary Search Tree is a binary tree where every node follows one simple rule: values smaller than the node go to its left subtree, and values greater than the node go to its right subtree.

The name says exactly what it does. It is a binary tree because each node can have at most two children, and it is built for searching because the ordering rule tells us which side to ignore.

Think of it like a sorted decision path. At each node, you ask one question: "Is my value smaller or larger?" That answer decides the next direction.

💡 A balanced BST can find values much faster than a linked list because it does not need to check every node one by one.
A BST keeps smaller values left and larger values right

Structure of a BST

A BST is made of nodes. Each node stores a value and two links: one to the left child and one to the right child. A group of nodes below any node is called a subtree.

BST Properties

The BST rule is not applied only at the root. It must be true for every node in the tree. That is what makes searching reliable.

1 All values in the left subtree are smaller than the current node.
2 All values in the right subtree are greater than the current node.
3 Both left and right subtrees must also be valid BSTs.

How Searching Works

Searching in a BST is a guided walk. Start at the root, compare the target with the current node, and move only in the direction where the answer can exist.

Search

Purpose

Find whether a value exists in the tree.

Idea

Compare once per level. Smaller goes left, larger goes right, equal means found.

Time: O(h)
Example: Search 60
50 → 70 → 60
60 is greater than 50, then smaller than 70, so the search turns right then left.
1Start at the root.
2If key is smaller, go left.
3If key is larger, go right.
4Stop when found or when the pointer becomes NULL.

How Insertion Works

Insertion works like searching, but instead of stopping at an existing value, we stop at an empty child pointer. The new node is attached there.

Insert

Purpose

Add a new value without breaking the BST rule.

Idea

Follow comparisons until the correct left or right child is empty.

How Deletion Works

Deletion must remove a value and still keep the BST valid. The simplest case is deleting a leaf node, because no child pointers need to be preserved.

BST Traversals

Traversal means visiting every node exactly once. A BST can use the same traversals as any binary tree, but one traversal becomes especially powerful: inorder traversal prints values in sorted order.

Traversal example tree
Inorder Left → Root → Right

Output: 20 → 30 → 40 → 50 → 60 → 70 → 80

Best takeaway: sorted order.
Preorder Root → Left → Right

Output: 50 → 30 → 20 → 40 → 70 → 60 → 80

Useful for copying tree structure.
Postorder Left → Right → Root

Output: 20 → 40 → 30 → 60 → 80 → 70 → 50

Useful for freeing/deleting a tree.
08

Implementation in C

The C implementation follows the same idea as the theory: compare with the current node, then move left or right. Below are the basic building blocks.

C — Node and Creation 
#include<stdio.h>
#include<stdlib.h>

struct node {
    int data;
    struct node* left;
    struct node* right;
};

struct node* createNode(int value) {
    struct node* newNode = (struct node*)malloc(sizeof(struct node));
    newNode->data = value;
    newNode->left = NULL;
    newNode->right = NULL;
    return newNode;
}
C — Basic Search Function 
struct node* search(struct node* root, int key) {
    if (root == NULL || root->data == key) {
        return root;
    }

    if (key < root->data) {
        return search(root->left, key);
    }

    return search(root->right, key);
}
C — Basic Insertion 
struct node* insert(struct node* root, int value) {
    if (root == NULL) {
        return createNode(value);
    }

    if (value < root->data) {
        root->left = insert(root->left, value);
    } else if (value > root->data) {
        root->right = insert(root->right, value);
    }

    return root;
}
👁️ Notice how search and insert both use the same decision rule: smaller values move left, larger values move right.
09

Time Complexity Analysis

BST operation time depends on the height of the tree. If the tree stays balanced, height is close to log n. If it becomes skewed, height can become n.

Operation Best Case Average Case Worst Case Reason
Search O(log n) O(log n) O(n) Follows one path from root to leaf.
Insert O(log n) O(log n) O(n) Searches for the empty position first.
Delete O(log n) O(log n) O(n) Finds the node, then repairs links.
Traversal O(n) O(n) O(n) Every node must be visited once.
Balanced BST Height is small, so search paths are short.
Skewed BST Tree behaves like a linked list, so performance drops.
10

BST vs Binary Tree

A BST is a binary tree with an extra ordering rule. That one rule changes how searching, traversal, and insertion behave.

Topic Binary Tree Binary Search Tree
Ordering No required value order. Left values are smaller; right values are larger.
Searching May need to check many nodes. Can skip an entire side at each comparison.
Insertion Depends on the tree's purpose. Inserted according to comparisons.
Efficiency Usually O(n) for search. O(log n) when balanced, O(n) when skewed.
Structure Only the "up to two children" rule exists. Every subtree must also follow BST order.

Advantages

⚡

Fast Searching

A balanced BST avoids unnecessary branches and reaches the value quickly.

↕

Ordered Data

Inorder traversal naturally gives values in sorted order.

＋

Dynamic Structure

Nodes are created as needed, so size can grow at runtime.

⤴

Efficient Insertion

New values are placed by following a single search path.

12

Limitations

!

Can become skewed

If values are inserted in sorted order, the BST can lean to one side.

↻

Recursive complexity

Many BST algorithms are recursive, so beginners must understand call flow clearly.

n

Worst-case O(n)

When the tree is skewed, search, insert, and delete may act like linked-list operations.

💡 Balanced BST variants like AVL Trees and Red-Black Trees solve this by rotating nodes to keep height small.
