Binary Trees Tree Data Structure

01

What is a Binary Tree?

🌲
A Binary Tree is a special type of tree where every node has at most 2 children — called the left child and the right child. Each node stores data plus two pointers. If a child doesn't exist, that pointer is NULL.

Unlike a general tree where a node can have any number of children, binary trees enforce a maximum of two. This constraint is what gives binary trees their power — it enables efficient searching, insertion, and recursive algorithms. The left and right distinction also allows ordering rules (like in BSTs) to be applied naturally.

💡 Every node in a binary tree holds three things: its data value, a left pointer (to its left subtree), and a right pointer (to its right subtree). Either or both pointers can be NULL.

02

Binary Tree Node Structure

Node memory layout — 3 fields per node

Each node in a binary tree has exactly 3 parts: a data field, a left pointer, and a right pointer. In C, this is defined as a struct.

C 
struct node {
    int data;
    struct node* left;
    struct node* right;
};
03

Types of Binary Trees

Binary trees come in several well-known forms, each with specific structural rules. Understanding these types is essential — they form the basis for BSTs, heaps, and balanced tree algorithms.

Full Binary Tree

Every node has 0 or 2 children — never exactly 1.

✔ Key: No node has exactly 1 child

Complete Binary Tree

All levels fully filled except the last, which fills left to right.

✔ Key: Last level fills strictly left-to-right

Perfect Binary Tree

All internal nodes have exactly 2 children; all leaves at the same level.

✔ Key: All leaves at same level; 2ʰ⁺¹ − 1 total nodes

Skewed Binary Tree

All nodes lean one direction — degenerates into a linked list.

✔ Key: O(n) search — no benefit over linked list

Balanced Binary Tree

For every node, the height difference between left and right subtrees is at most 1.

⚖️ This balance constraint keeps height at O(log n), ensuring fast search and insertion. AVL Trees and Red-Black Trees are self-balancing binary trees that maintain this property automatically.

✔ Key: |height(left) − height(right)| ≤ 1 at every node

04

Binary Tree Properties

📐

Max nodes at Level L = 2L

Level 0 (root) = 1 node. Level 1 = 2. Level 2 = 4. Each level doubles.

🌲

Max nodes in tree of height H = 2H+1 − 1

A perfect binary tree of height 2 has 7 nodes: 2³ − 1 = 7.

📏

Min height for N nodes = ⌊log₂N⌋

7 nodes need height at least 2. Keeping height minimal = fast operations.

🔗

N nodes → exactly N−1 edges

Every node except root has exactly one incoming edge. So edges = nodes − 1.

🍃

Max leaves at height H = 2H

At height 2, a perfect tree has 4 leaf nodes at the bottom level.

LEVEL FORMULA
Level 0 2⁰ = 1 node
Level 1 2¹ = 2 nodes
Level 2 2² = 4 nodes
Total = 7 = 2³ − 1 ✓

05

Creating a Binary Tree in C

✨
Visualize Code
Watch step-by-step execution with animations
→

A binary tree is built recursively using a create() function. The user inputs data for each node one by one. Entering -1 means no node at that position — the function returns NULL. The left subtree is fully built before the right subtree.

C 
#include<stdio.h>
#include<stdlib.h>

struct node{
    int data;
    struct node* left, *right;
};

struct node* create(){
    int x;
    printf("Enter data (Enter -1 to stop or if no data) : ");
    scanf("%d", &x);
    if(x == -1){
        return NULL;
    }
    struct node* newnode = malloc(sizeof(struct node));
    newnode->data = x;
    newnode->left = create();
    newnode->right = create();
    return newnode;
}

int main(){
    struct node* root = create();
}
Recursion trace — input sequence builds the tree step by step
🔁 The recursion goes left-first: it fully builds the left subtree of a node before asking for the right child. Entering -1 is how you tell the function "no node here" — it returns NULL immediately without creating a struct.
06

Advantages

🗂️

Natural Hierarchical Structure

Binary trees mirror real-world hierarchies — file systems, org charts, and expression trees all map directly to this structure.

🏗️

Foundation for Advanced Structures

BST, AVL Tree, Heap, Trie — everything builds on binary trees. Mastering this unlocks the entire upper half of data structures.

🔁

Efficient Recursive Algorithms

Tree problems decompose naturally into "process root, recurse left, recurse right" — making recursive solutions clean and elegant.

🧮

Expression Trees & Compilers

Compilers use binary trees to represent and evaluate arithmetic expressions. Operators are internal nodes, operands are leaves.

07

Limitations

❌

No Built-in Ordering

A basic binary tree has no rule about where values go. You need a BST (with left < root < right) to enable fast searching.

📉

Skewing Degrades Performance

Without balancing, a tree can become skewed — all nodes on one side. Search degrades from O(log n) to O(n), no better than a linked list.

🧠

Pointer Overhead

Each node stores two pointers (left + right) in addition to data. For large trees, this extra memory adds up compared to arrays.

⚙️

More Complex Than Linear Structures

Traversal, insertion, and deletion all require recursive thinking. Harder to implement and debug than arrays or linked lists.
