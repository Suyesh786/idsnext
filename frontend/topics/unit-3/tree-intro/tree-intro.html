Trees & Terminology Hierarchical Data Structure

01

What is a Tree?

🌳
A Tree is a non-linear, hierarchical data structure made of nodes connected by edges. Unlike arrays or linked lists that are linear (one item after another), trees branch out — each node can have multiple children below it.

Think of a family tree: a grandparent at the top, two parents below, and grandchildren further down. Your computer's file system is a tree — folders contain sub-folders and files. An org chart is a tree — the CEO at the top, departments branching below. Trees model these real-world hierarchies naturally.

💡 Trees are connected and acyclic — there are no loops. Every node (except the root) has exactly one parent, and there is exactly one path between any two nodes.

02

Why Trees?

🐌

Linear Search is Slow

Arrays and linked lists require visiting every element to find one. That's O(n) — with 1 million items, up to 1 million checks.

⚡

Trees Enable O(log n)

A balanced Binary Search Tree halves the search space at each step. 1 million items → only ~20 comparisons needed.

🗂️

Hierarchical Data Fits Naturally

File systems, HTML DOM, org charts, menus — all have parent-child structure. Trees model this directly; arrays cannot.

03

Tree Terminology

Every term you need — mapped to the diagram. Study it first, then read each definition.

Node

Basic unit of a tree. Holds a data value and references to its children.

Root

Topmost node with no parent. Every tree has exactly one root.

Edge

A link connecting a parent node to a child node. N nodes → N−1 edges.

Parent

A node that has one or more children directly below it.

Child

A node connected directly below a parent node. Multiple children possible.

Leaf

Node with no children. Also called an external node. Always at the bottom.

Siblings

Nodes that share the same parent. D and E are siblings if both children of B.

Subtree

Any node plus all its descendants forms a subtree — itself a valid tree.

Height

Longest path from root down to any leaf node. Measured in edges.

Depth

Distance (in edges) from the root to a specific node. Root has depth 0.

Level

All nodes at the same depth form a level. Root is Level 0.

Degree

Number of children a node has. Leaf nodes have degree 0.

04

Tree Properties

🔢

N nodes → N−1 edges

A tree with N nodes always has exactly N−1 edges. No more, no fewer.

🛤️

Unique path between any two nodes

In a tree, there is one and only one path from any node to any other node.

👑

Every node (except root) has exactly one parent

The root is the only node without a parent. All others have exactly one.

🔗

Trees are connected and acyclic

All nodes are reachable from the root, and there are no cycles (loops).

∅

Empty tree is valid

A tree with zero nodes is a valid, legal tree — just an empty one.

📐

Height ≥ log₂(N) for balanced trees

A perfectly balanced binary tree keeps height minimal — key to fast operations.

05

Types of Trees (overview — detail in upcoming topics)

Trees come in many flavors, each optimized for different use cases. Here's a quick preview — we'll deep-dive into each in upcoming topics.

Binary Tree

Each node has at most 2 children — left and right. The simplest tree structure.

Binary Search Tree (BST)

Left child < parent < right child. Enables O(log n) search, insert, and delete.

AVL Tree

A self-balancing BST. Automatically rebalances after insert/delete to keep height O(log n).

Expression Tree

Used to represent math expressions. Operators at internal nodes, operands at leaves.

07

Real-World Applications

📁

File Systems

Folders contain sub-folders and files in a tree. Every OS uses a directory tree — /home/user/documents is a path from root to leaf.

🌐

HTML DOM

A webpage's structure is a tree. <html> is the root, <body> and <head> are children, nested tags are deeper nodes.

🏢

Organization Charts

CEO at the root, departments as internal nodes, individual employees as leaf nodes. Classic tree structure.

🤖

Decision Trees (AI / ML)

Machine learning models make predictions by branching on yes/no questions — each question is an internal node, each outcome a leaf.

🗄️

Database Indexing (B-Trees)

Databases use B-Trees to index billions of rows. Searching for a record takes O(log n) instead of scanning the full table.

🔤

Autocomplete / Tries

Search bars use Trie trees to suggest completions. Each character is an edge; all words sharing a prefix share tree branches.
