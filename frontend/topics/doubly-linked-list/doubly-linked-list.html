Doubly Linked List Linear – Bi-directional Access

01

What is a Doubly Linked List?

🔗
A Doubly Linked List is a linear data structure in which elements, called nodes, are connected in both directions. Each node holds a data field, a next pointer referencing the following node, and a prev pointer referencing the preceding node — the head's prev and the tail's next are both NULL.

Unlike a singly linked list, a doubly linked list maintains two pointer fields per node. This allows traversal in both forward (head → tail) and backward (tail → head) directions. Deletions become O(1) once the target node is known — no need to track the predecessor separately, since prev already holds that reference.

🚇

Subway Train

Each carriage connects to the one ahead and the one behind. You can walk forward or backward through the train freely — bidirectional movement is built in.

📖

Book Pages

Each page links to the next and previous page. You can flip forward or flip backward without returning to the cover — the prev pointer is the page number you just came from.

🏃

Two-Way Street

A singly linked list is a one-way street. A doubly linked list adds a return lane — you know both where to go next and where you just came from at every node.

Every insert must update both next and prev pointers to maintain bidirectional linkage.

C — Insert Operations 
// Node structure
struct Node {
    int data;
    struct Node* prev;
    struct Node* next;
};

struct Node* head = NULL;
struct Node* tail = NULL;

// Insert at beginning — O(1)
void insertBegin(int value) {
    struct Node* n = malloc(sizeof(struct Node));
    n->data = value;
    n->prev = NULL;
    n->next = head;
    if (head != NULL) head->prev = n;
    else tail = n;  // list was empty
    head = n;
}

// Insert at end — O(1) with tail pointer
void insertEnd(int value) {
    struct Node* n = malloc(sizeof(struct Node));
    n->data = value;
    n->next = NULL;
    n->prev = tail;
    if (tail != NULL) tail->next = n;
    else head = n;  // list was empty
    tail = n;
}

// Insert at position (0-based) — O(n)
void insertAt(int pos, int value) {
    if (pos == 0) { insertBegin(value); return; }

    struct Node* prev = head;
    for (int i = 0; i < pos - 1 && prev != NULL; i++)
        prev = prev->next;

    if (prev == NULL) { printf("Invalid position.\n"); return; }
    if (prev->next == NULL) { insertEnd(value); return; }

    struct Node* n = malloc(sizeof(struct Node));
    n->data  = value;
    n->next  = prev->next;      // step 1
    n->prev  = prev;            // step 2
    prev->next->prev = n;       // step 3
    prev->next = n;             // step 4
}
ℹ️ Four-pointer rule: Middle insertion always requires exactly 4 pointer updates — two on the new node and one each on its predecessor and successor.

Because each node knows its predecessor via prev, deletion is O(1) once the node reference is available — unlike SLL which requires predecessor traversal.

C — Delete Operations 
// Delete at beginning — O(1)
void deleteBegin() {
    if (head == NULL) return;
    struct Node* temp = head;
    head = head->next;
    if (head != NULL) head->prev = NULL;
    else tail = NULL;
    free(temp);
}

// Delete at end — O(1) with tail pointer
void deleteEnd() {
    if (tail == NULL) return;
    struct Node* temp = tail;
    tail = tail->prev;
    if (tail != NULL) tail->next = NULL;
    else head = NULL;
    free(temp);
}

// Delete by value — O(n) search, O(1) unlink
void deleteByValue(int value) {
    struct Node* cur = head;
    while (cur != NULL && cur->data != value)
        cur = cur->next;

    if (cur == NULL) { printf("Value not found.\n"); return; }

    // Unlink — no predecessor search needed!
    if (cur->prev != NULL) cur->prev->next = cur->next;
    else head = cur->next;   // cur was head

    if (cur->next != NULL) cur->next->prev = cur->prev;
    else tail = cur->prev;   // cur was tail

    free(cur);
}
ℹ️ DLL advantage: deleteEnd() is O(1) using the tail pointer and prev. In a singly linked list, deleteEnd() is O(n) because you must traverse to find the new tail.

A doubly linked list can be traversed in both directions. Forward traversal uses next; backward traversal uses prev. Both are O(n).

C — Display & Traversal 
// Forward traversal — O(n)
void displayForward() {
    struct Node* cur = head;
    printf("NULL ← HEAD → ");
    while (cur != NULL) {
        printf("[%d] ⇆ ", cur->data);
        cur = cur->next;
    }
    printf("NULL\n");
}

// Backward traversal — O(n) — unique to DLL!
void displayBackward() {
    struct Node* cur = tail;
    printf("NULL ← TAIL → ");
    while (cur != NULL) {
        printf("[%d] ⇆ ", cur->data);
        cur = cur->prev;
    }
    printf("NULL\n");
}
ℹ️ Backward traversal: This is impossible in a singly linked list without reversing the list first. The DLL's prev pointer makes tail→head traversal trivial and O(n).

04

Advantages & Limitations

✅

Bidirectional Traversal

Traverse the list in either direction at any time. Enables efficient algorithms that need backward scanning, like LRU cache eviction.

✅

O(1) Delete at Tail

The prev pointer lets you remove the tail node in O(1) — no traversal needed. Singly linked lists require O(n) for the same operation.

✅

Efficient Unlink

Given a node reference, unlinking it from the list requires only O(1) work — no predecessor search, just pointer reassignment on both sides.

❌

Extra Memory

Every node carries an additional prev pointer — 4 or 8 extra bytes. For millions of small nodes, this doubles pointer overhead vs SLL.

❌

More Complex Code

Every insert and delete must correctly update two pointers instead of one. Missing any update corrupts the list in both directions.

❌

No Random Access

Like SLL, accessing element at index i still requires O(n) traversal from head (or O(n/2) from the closer end). No O(1) indexing.

05

Applications of Doubly Linked List

↩️

Browser History

The back/forward navigation in a browser is a classic DLL. Each page is a node; the back button follows prev, the forward button follows next.

🗃️

LRU Cache

Least Recently Used cache combines a hash map with a DLL. The DLL lets O(1) removal of any node (most/least recently used) without traversal.

🎵

Music Playlist

Each track is a node. Previous/Next song navigation uses prev and next pointers — loop-back is easy by connecting tail back to head.

📝

Undo / Redo

Text editor undo/redo stacks are implemented as DLLs. Each state is a node; undo follows prev, redo follows next.
