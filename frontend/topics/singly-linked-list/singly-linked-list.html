Singly Linked List Linear – Sequential Access

01

What is a Singly Linked List?

🔗
A Singly Linked List is a linear data structure in which elements, called nodes, are connected sequentially. Each node holds a data field and a next pointer that references the following node — the last node's next pointer is NULL.

Unlike arrays, a singly linked list does not store elements in contiguous memory. Each node is allocated independently in the heap, and the chain is maintained purely by pointers. This gives the list dynamic sizing — it can grow or shrink at runtime without pre-allocation. Traversal is strictly one-directional: from the head node toward the tail.

🚂

Train Carriages

Each carriage (node) connects to the next one via a coupling (pointer). You board from the front (head) and walk carriage by carriage — you cannot jump to the middle.

📎

Paper Clip Chain

Each clip holds data and links to the next clip. Adding a clip at either end is fast, but finding a specific one means scanning from the first clip forward.

🌐

Scavenger Hunt

Each clue (node) reveals the location of the next clue (next pointer). You must start at clue #1 (head) and follow the chain — there is no random access to clue #5.

Operations

Insert Begin

Create a new node, set its next to current head, then update head to the new node.

Time Complexity O(1)
Insert End

With a tail pointer: create node, set tail's next, update tail. No traversal needed.

Time Complexity O(1)
Delete Node

Locate the target node's predecessor, redirect its next pointer, then free the node.

Time Complexity O(n)
Traversal

Start at head, visit each node by following next pointers until NULL is reached.

04

Implementation in C

The foundational building block — a Node struct with a data field and a self-referential pointer to the next node.

C — Node Structure 
#include <stdio.h>
#include <stdlib.h>

// Node structure definition
struct Node {
    int data;            // Data stored in the node
    struct Node* next;   // Pointer to next node
};

// Head and tail pointers
struct Node* head = NULL;
struct Node* tail = NULL;

// Helper: create a new node
struct Node* createNode(int value) {
    struct Node* newNode =
        (struct Node*) malloc(sizeof(struct Node));
    newNode->data = value;
    newNode->next = NULL;
    return newNode;
}
ℹ️ The next pointer inside struct Node is self-referential — it points to another object of the same type. This is what forms the chain.

Insert at the beginning runs in O(1). Insert at the end also runs in O(1) because we maintain a tail pointer — no traversal required.

C — Insert at Beginning & End 
// Insert at beginning — O(1)
void insertAtBeginning(int value) {
    struct Node* newNode = createNode(value);
    if (head == NULL) {
        head = tail = newNode;  // First node
    } else {
        newNode->next = head;
        head = newNode;
    }
}

// Insert at end — O(1) with tail pointer
void insertAtEnd(int value) {
    struct Node* newNode = createNode(value);
    if (tail == NULL) {
        head = tail = newNode;  // First node
    } else {
        tail->next = newNode;
        tail = newNode;
    }
}

// Delete by value — O(n)
void deleteNode(int value) {
    if (head == NULL) { printf("List is empty.\n"); return; }

    // Deleting head node
    if (head->data == value) {
        struct Node* temp = head;
        head = head->next;
        if (head == NULL) tail = NULL;
        free(temp);
        return;
    }

    // Find predecessor
    struct Node* prev = head;
    while (prev->next != NULL && prev->next->data != value)
        prev = prev->next;

    if (prev->next == NULL) { printf("Value not found.\n"); return; }

    struct Node* target = prev->next;
    prev->next = target->next;
    if (target == tail) tail = prev;
    free(target);
}
ℹ️ Tail pointer advantage: Without a tail pointer, inserting at the end requires traversing the entire list — O(n). Maintaining tail reduces it to O(1).

Traversal starts at the head and visits each node by following the next pointer until NULL is reached. This always takes O(n) time.

C — Display & Traversal 
// Display all nodes — O(n)
void display() {
    if (head == NULL) {
        printf("List is empty.\n");
        return;
    }
    struct Node* current = head;
    printf("HEAD → ");
    while (current != NULL) {
        printf("[%d | next] → ", current->data);
        current = current->next;
    }
    printf("NULL\n");
}

// Count nodes — O(n)
int length() {
    int count = 0;
    struct Node* current = head;
    while (current != NULL) {
        count++;
        current = current->next;
    }
    return count;
}

// Search for a value — O(n)
int search(int value) {
    struct Node* current = head;
    int pos = 0;
    while (current != NULL) {
        if (current->data == value) return pos;
        current = current->next;
        pos++;
    }
    return -1;  // Not found
}
ℹ️ No random access: Unlike arrays, you cannot jump to index i in O(1). You must traverse from head, which makes singly linked lists unsuitable for binary search.

05

Advantages & Limitations

✅

Dynamic Size

Memory is allocated on demand. The list expands or contracts at runtime with no wasted pre-allocated space.

✅

Fast Head Insertion

Inserting at the beginning is O(1) — simply update the head pointer. No shifting of existing elements required.

✅

Efficient Deletion

Once the predecessor is found, deletion is O(1) — just relink pointers. Arrays would require shifting all subsequent elements.

❌

No Random Access

Accessing element at position i requires traversal from head — O(n). Arrays access any element in O(1) via index.

❌

Extra Memory

Every node stores an additional pointer field (4 or 8 bytes). For small data types, this overhead can be significant.

❌

No Reverse Traversal

Singly linked lists can only be traversed forward. Backward traversal requires a doubly linked list or a full reversal pass.

06

Applications of Singly Linked List

🧠

Dynamic Memory Allocation

OS free-block lists are implemented as linked lists. When memory is freed, its block is inserted; when allocated, a suitable block is removed from the list.

📚

Implementing Stacks & Queues

Both stacks and queues can be built on a singly linked list. Stack uses head insertion/deletion; Queue uses tail insertion and head deletion.

🗺️

Graph Adjacency Lists

Each vertex in a graph stores its neighbours as a singly linked list. This representation is memory-efficient for sparse graphs compared to an adjacency matrix.

🔢

Polynomial Representation

Sparse polynomials are stored as linked lists where each node holds a coefficient and exponent pair — efficient addition and multiplication without wasted zero terms.
