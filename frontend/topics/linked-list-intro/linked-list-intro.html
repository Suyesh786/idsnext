Linked Lists Node-based Dynamic Structure

01

What is a Linked List?

🔗
A Linked List is a linear data structure where elements, called nodes, are stored in memory non-contiguously. Each node holds a data value and a pointer to the next node in the sequence.

Unlike arrays, linked lists do not store elements in consecutive memory locations. Instead, each node can live anywhere in memory — they are connected by pointers. This makes linked lists inherently dynamic: their size can grow or shrink at runtime without needing to declare a fixed capacity upfront.

💡 The first node in a linked list is called the head. The last node points to NULL, signalling the end of the list.
02

Why Linked Lists?

📦

Array Limitation

Arrays have a fixed size declared at compile time. If you need more space, you must reallocate and copy — an expensive O(n) operation.

⚡

Dynamic Allocation

Linked lists grow and shrink on demand. Memory is requested from the heap only when a new node is needed — no wasted pre-allocation.

🔄

Efficient Insert / Delete

Inserting or deleting a node only requires updating a couple of pointers — O(1) at head or tail — unlike arrays which require shifting elements.

03

Structure of a Node

Every linked list is made of nodes. A node has exactly two fields: a data field that stores the value, and a next pointer that stores the address of the following node.

data 42
next →
data — stores the actual value (int, char, struct, etc.)
next — pointer to the next node in the list (NULL if last)
C — Node Structure 
struct Node {
    int          data;   // stores the value
    struct Node* next;   // pointer to next node
};

// Create a new node dynamically
struct Node* createNode(int value) {
    struct Node* newNode =
        (struct Node*) malloc(sizeof(struct Node));
    newNode->data = value;
    newNode->next = NULL;
    return newNode;
}
ℹ️ malloc() allocates memory on the heap. Always free() nodes when they are no longer needed to prevent memory leaks.

05

Advantages & Limitations

✅

Advantages

  • Dynamic size — grows and shrinks at runtime with no pre-allocation needed.
  • Efficient insertion/deletion — O(1) at head or when pointer is known.
  • No memory waste — allocate only what you need, when you need it.
  • Flexible memory — nodes need not be stored contiguously.
⚠️

Limitations

  • No random access — to reach element at index n, you must traverse from head. O(n).
  • Extra memory — every node stores a pointer in addition to data.
  • Cache unfriendly — non-contiguous storage leads to more cache misses than arrays.
  • Reverse traversal — not possible with singly linked lists without extra work.

06

Applications of Linked Lists

🗃️

Dynamic Data Storage

Whenever the number of elements is unknown at compile time — user inputs, file records, runtime collections — linked lists manage memory efficiently without over-allocation.

📚

Implementing Stacks & Queues

Stacks and queues are typically built on top of linked lists. Push/enqueue adds a node; pop/dequeue removes one — both O(1) without array resizing overhead.

🗺️

Graph Adjacency Lists

Graphs with sparse edges are represented using adjacency lists — each vertex holds a linked list of its neighbours, saving memory over adjacency matrices.

↩️

Undo / Redo History

Text editors and design tools maintain a doubly linked list of states. Moving forward or backward through history is simply following the next or prev pointer.
