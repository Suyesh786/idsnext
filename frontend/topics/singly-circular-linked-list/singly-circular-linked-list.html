Singly Circular Linked List Circular – One-directional

01

What is a Singly Circular Linked List?

🔄
A Singly Circular Linked List is a variation of the singly linked list where the last node's next pointer points back to the first node (head) instead of NULL, forming a continuous circular loop.

Unlike a standard singly linked list there is no NULL terminator. Traversal wraps around — once you reach the last node, the next step takes you back to head. This is ideal for round-robin scheduling, circular buffers, and any cyclic process.

🎡

Ferris Wheel

Each cabin connects to the next and the last cabin connects back to the first — the ride loops endlessly with no defined end.

🔁

Playlist Loop

Songs play one after another; when the last song ends it loops back to the first — exactly how a circular linked list cycles.

🏃

Round-Robin

CPU scheduling gives each process a time slice. After the last process runs, the scheduler wraps to the first — circular structure in action.

Key Properties

No NULL — tail's next always points to head
One-directional — head → tail → head
Dynamic — grows/shrinks at runtime
Stop guard: stop traversal when curr == head

vs. Singly Linked List

SLL: tail → NULL  |  SCLL: tail → head
SLL: stops at NULL  |  SCLL: stops at head
SCLL enables continuous cycling without reset
03

Operations

Insert Begin

Create node, set its next to current head, update tail's next to new node, update head.

TimeO(1)
Insert End

Create node, link tail to new node, set new node's next to head, update tail.

TimeO(1)
Delete Node

Find predecessor, relink its next, maintain circular invariant (no NULL).

TimeO(n)
Traversal

Start at head with do-while; stop when current == head again.

TimeO(n)
04

Implementation in C

The node structure is identical to SLL — circularity lives entirely in pointer assignments.

C — Node Structure 
#include <stdio.h>
#include <stdlib.h>

struct Node {
    int data;
    struct Node* next;
};

struct Node* head = NULL;
struct Node* tail = NULL;

struct Node* createNode(int value) {
    struct Node* n = (struct Node*)malloc(sizeof(struct Node));
    n->data = value;
    n->next = NULL;
    return n;
}

Every operation must preserve tail->next == head.

C — Insert & Delete 
void insertAtBeginning(int val) {
    struct Node* n = createNode(val);
    if (head == NULL) { head = tail = n; n->next = head; }
    else { n->next = head; head = n; tail->next = head; }
}

void insertAtEnd(int val) {
    struct Node* n = createNode(val);
    if (head == NULL) { head = tail = n; n->next = head; }
    else { tail->next = n; tail = n; tail->next = head; }
}

void deleteNode(int val) {
    if (head == NULL) { printf("Empty.\n"); return; }
    struct Node* curr = head, *prev = NULL;
    do {
        if (curr->data == val) break;
        prev = curr; curr = curr->next;
    } while (curr != head);

    if (curr->data != val) { printf("Not found.\n"); return; }
    if (curr == head && curr == tail) { head = tail = NULL; }
    else if (curr == head) { head = head->next; tail->next = head; }
    else if (curr == tail) { prev->next = head; tail = prev; }
    else { prev->next = curr->next; }
    free(curr);
}
ℹ️ Circular invariant: after every operation confirm tail->next == head. Breaking it causes an infinite traversal loop.

Use do-while — a plain while(curr!=NULL) loops forever.

C — Circular Traversal 
void display() {
    if (head == NULL) { printf("Empty.\n"); return; }
    struct Node* curr = head;
    printf("HEAD → ");
    do {
        printf("[%d] → ", curr->data);
        curr = curr->next;
    } while (curr != head);
    printf("(back to HEAD)\n");
}

05

Advantages & Limitations

✅

Infinite Cycling

Traversal wraps naturally — perfect for round-robin schedulers and circular buffers.

✅

Queue Implementation

Can implement a queue efficiently using a single tail pointer — tail->next is always the front.

❌

Traversal Risk

Plain while(curr!=NULL) causes an infinite loop. Must always use do-while terminating at head.

❌

No Reverse Access

One-directional only. Use a Doubly Circular LL for bidirectional traversal.

06

Applications

🖥️

CPU Round-Robin Scheduling

OS schedulers cycle through processes in a circular list, giving each a time slice before wrapping back to the first.

🎵

Music Playlist Loop

Looped playlists cycle indefinitely — the last song's next points back to the first song node.

🎮

Multiplayer Game Turns

Player turns cycle circularly — after the last player, control wraps to the first automatically.

📡

Token Ring Networks

A token circulates through network nodes in a ring — the circular list models this perfectly.
