Singly Circular Linked List Circular – Infinite Traversal

01

What is a Singly Circular Linked List?

🔄
A Singly Circular Linked List is a variation of a linked list where each node has a data field and a next pointer — but unlike a regular linked list, the last node points back to the head instead of pointing to NULL. This forms a continuous loop.

The key difference from a regular singly linked list is the circular nature: traversal never reaches a dead end. Once you reach the last node, the next pointer brings you back to the beginning — creating an infinite loop if not handled carefully. This property makes it ideal for cyclic systems like round-robin schedulers and music playlists.

Circular Structure
1 → 2 → 3 → 4
↑ ← ← ← ← ← ← ↓
Last node (4) points back to head (1) — no NULL!
🎡

Ferris Wheel

Cabins keep rotating in a circle. After the last cabin comes the first one again — it never stops at a dead end, just like a circular linked list.

🎵

Repeat Playlist

Songs play one after another. When the last song ends, it wraps back to the first. Each song is a node; the wrap-around is the circular pointer.

⏱️

Round Robin

CPU scheduler gives each process a turn. After the last process, it goes back to the first — a perfect real-world use of circular linked lists.

02

Node Structure

Each node in a singly circular linked list has exactly two fields — same as a regular singly linked list. The only behavioral difference is what the next pointer of the last node stores.

Node Structure
data 42
next → head
Last node's next points to head — not NULL
C — Node Definition
struct node {
    int data;          // stores the value
    struct node* next; // points to next node (last → head)
};
📦

data

Stores the actual value — integer, character, or any type. This is the payload the node carries.

➡️

next pointer

Holds the address of the next node. For the last node, this points back to the head — closing the circle.

🚫

No prev pointer

Unlike a doubly circular list, there is no previous pointer. Traversal is one direction only — forward.

04

Key Characteristics

Singly circular linked lists have unique properties that set them apart from both regular linked lists and other data structures. Understanding these characteristics helps you know when — and when not — to use them.

∞

No NULL Pointer

No node in the list points to NULL. The last node always points back to the head, completing the circle.

🔁

Forms a Loop

All nodes form one continuous loop. You can start at any node and eventually reach every other node by following next pointers.

⚠️

Infinite Traversal Risk

A naive while loop checking for NULL will run forever. You must use a do-while loop or stop condition based on the head node.

🎯

Any Node as Start

Since the list is circular, you can technically start traversal from any node and still visit all nodes exactly once.

05

Traversal

Traversal in a circular linked list requires special attention. Because there is no NULL, the standard while-loop approach used for singly linked lists will cause an infinite loop.

❌ Wrong — Infinite Loop!
// WRONG: temp never becomes NULL!
struct node* temp = head;
while(temp != NULL) {
    printf("%d ", temp->data);
    temp = temp->next; // loops forever
}

Since no node's next is NULL, this loop never terminates. Avoid this pattern!

✔ Correct — do-while Loop
// CORRECT: stop when we reach head again
struct node* temp = head;
do {
    printf("%d ", temp->data);
    temp = temp->next;
} while(temp != head);

The do-while ensures we visit the head at least once, then stop when we circle back to it.

06

Insertion Operations

Inserting into a circular linked list requires maintaining the circular link at all times. The trickiest part is always ensuring the last node's next points back to the new (or existing) head.

Insert at Beginning ▾
  1. Create a new node with the given data.
  2. If the list is empty, point the new node's next to itself — it becomes both head and tail.
  3. If not empty, traverse to find the last node (the one whose next == head).
  4. Set new node's next = current head.
  5. Set last node's next = new node (so the circle is maintained).
  6. Update head = new node.
void insertBeginning(struct node** head, int data) {
    struct node* newNode = createNode(data);
    if (*head == NULL) {
        newNode->next = newNode; // points to itself
        *head = newNode;
        return;
    }
    // find last node
    struct node* last = *head;
    while(last->next != *head)
        last = last->next;
    newNode->next = *head;
    last->next = newNode; // maintain circle
    *head = newNode;
}
Insert at End ▾
  1. Create a new node.
  2. If list is empty, new node points to itself and becomes head.
  3. Traverse to the last node (whose next == head).
  4. Set last node's next = new node.
  5. Set new node's next = head (completes the circle).
void insertEnd(struct node** head, int data) {
    struct node* newNode = createNode(data);
    if (*head == NULL) {
        newNode->next = newNode;
        *head = newNode; return;
    }
    struct node* last = *head;
    while(last->next != *head)
        last = last->next;
    last->next = newNode;
    newNode->next = *head; // close the circle
}
Insert at Position ▾
  1. If position is 0, use Insert at Beginning.
  2. Traverse to the node just before the desired position.
  3. Create new node, set its next = node at position.
  4. Set previous node's next = new node.
  5. If inserting after the last node, also update last node's next = head.
void insertAtPos(struct node** head, int data, int pos) {
    if (pos == 0) { insertBeginning(head, data); return; }
    struct node* temp = *head;
    for(int i = 0; i < pos - 1; i++) {
        temp = temp->next;
        if(temp == *head) break; // guard: past end
    }
    struct node* newNode = createNode(data);
    newNode->next = temp->next;
    temp->next = newNode;
}
07

Deletion Operations

Deletion must always preserve the circular structure. When deleting the head or tail node, you must update the last node's next pointer to maintain the loop.

Delete at Beginning ▾
  1. If list has only one node, set head = NULL and free the node.
  2. Traverse to find the last node (next == head).
  3. Set last node's next = head's next (skips current head).
  4. Update head = head's next.
  5. Free the old head node.
void deleteBeginning(struct node** head) {
    if(*head == NULL) return;
    struct node* last = *head;
    while(last->next != *head)
        last = last->next;
    if(*head == last) { // single node
        free(*head); *head = NULL; return;
    }
    struct node* temp = *head;
    last->next = (*head)->next;
    *head = (*head)->next;
    free(temp);
}
Delete at End ▾
  1. If list has only one node, free it and set head = NULL.
  2. Traverse to second-to-last node (whose next's next == head).
  3. Free the last node.
  4. Set second-to-last node's next = head (restores circle).
void deleteEnd(struct node** head) {
    if(*head == NULL) return;
    struct node* prev = *head;
    struct node* last = (*head)->next;
    if(last == *head) { // one node
        free(*head); *head = NULL; return;
    }
    while(last->next != *head) {
        prev = last;
        last = last->next;
    }
    prev->next = *head; // restore circle
    free(last);
}
Delete at Position ▾
  1. If position is 0, use Delete at Beginning.
  2. Traverse to the node just before the target position.
  3. Store the target node in a temporary pointer.
  4. Update previous node's next to skip the target node.
  5. Free the target node.
void deleteAtPos(struct node** head, int pos) {
    if(pos == 0) { deleteBeginning(head); return; }
    struct node* temp = *head;
    for(int i = 0; i < pos - 1; i++)
        temp = temp->next;
    struct node* toDel = temp->next;
    temp->next = toDel->next;
    free(toDel);
}

C — Complete Implementation
#include <stdio.h>
#include <stdlib.h>

struct node {
    int data;
    struct node* next;
};

struct node* head = NULL;

void insert(int data) {
    struct node* newnode = (struct node*)malloc(sizeof(struct node));
    newnode->data = data;

    if (head == NULL) {
        newnode->next = newnode;
        head = newnode;
        return;
    }

    struct node* temp = head;
    while (temp->next != head) {
        temp = temp->next;
    }

    temp->next = newnode;
    newnode->next = head;
}
09

Advantages

⚡

Efficient Looping

Cycling through elements repeatedly is natural — no need to reset the pointer back to head manually. Just keep following next.

🛡️

No NULL Checks

Since no node is ever NULL, you avoid NULL pointer dereference bugs during traversal. Every node is valid and connected.

🔄

Cyclic System Fit

Ideal for systems that inherently repeat — OS schedulers, media players, buffering — where the circular structure maps naturally to the problem.

10

Disadvantages

🐛

Hard to Debug

Circular references make it hard to print or inspect the list. Standard debugging tools may follow the loop indefinitely.

♾️

Infinite Loop Risk

A single mistake in the traversal condition can cause an infinite loop that freezes your program. Extra care with stopping conditions is essential.

🧩

More Complex Logic

Every insertion and deletion must carefully maintain the circular link. This is more complex than a standard singly linked list where you just set NULL.

11

Real-World Applications

The circular nature of this data structure makes it a perfect match for real-world cyclic problems. Here are the most common use cases:

🖥️

Round Robin Scheduling

Operating systems give each process equal CPU time in a cycle. The circular list lets the scheduler loop back to the first process after the last one — endlessly and fairly.

🎶

Music Playlists (Loop Mode)

When a playlist is set to repeat, each song node's next points to the following track. The last song's next points back to the first — exactly a circular linked list.

🗄️

Circular Buffers

Used in streaming, audio processing, and networking. Data is written and read in a circle — when you reach the end, you wrap around to the start, reusing space efficiently.

🎮

Turn-Based Games

Multiplayer games cycle through players' turns. A circular list of player nodes allows seamless looping — after the last player, it's the first player's turn again.
